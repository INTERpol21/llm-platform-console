# CLAUDE.md — llm-platform-console

Working notes for AI agents. Generated from a code-level survey (2026-07-23); verify paths still exist before relying on them.

Web console ("fifth project") over a four-service LLM platform (llm-gateway, rag-pgvector, mcp-tools-server, agent-orchestrator). A pnpm monorepo with three workspace packages: packages/contracts (committed OpenAPI snapshots + Kubb-generated TS types and Zod schemas), bff (Hono reverse proxy that injects platform bearer keys, rate-limits, and passes SSE through untouched), and web (React 19 Vite SPA in strict Feature-Sliced Design with CSS Modules and RU/EN i18n). docker-compose.yml is the umbrella that runs the whole platform — Postgres(pgvector)+Redis+4 backends+BFF+Caddy — behind a single origin :8080, fully offline with mock models by default.

## Commands
```bash
pnpm install                    # pnpm 11.15.1 via packageManager (corepack or npm i -g)
pnpm -r typecheck && pnpm -r test
pnpm exec biome check . && pnpm --filter @console/web lint:fsd
pnpm contracts                  # regen Kubb types from packages/contracts/openapi/*.json
docker compose up -d --build --wait   # umbrella: 8 containers, console on :8080
E2E_BASE_URL=http://localhost:8080 pnpm --filter @console/web test:e2e
```

## Layout
- `packages/contracts` — Contract source of truth. openapi/{gateway,rag,orchestrator}.json are real snapshots exported from the backend FastAPI apps; kubb.config.ts generates src/<service>/{types,zod,schemas} (one Kubb config per service, output clean:true). Exports subpaths @console/contracts/{gateway,rag,orchestrator}; the root barrel namespaces them so same-named schemas (HTTPValidationError) don't collide. Generated code IS committed — CI regenerates and fails on diff.
- `bff/src` — Hono BFF (port 8787). app.ts wires /api/health (pre-limiter), a bodyLimit cap over /api/* (BFF_MAX_BODY_BYTES, default 12 MiB, 413 before any proxying; counts streamed bytes so chunked uploads are covered too), a token-bucket rate limiter over /api/* (rate-limit.ts, in-memory, per-client via XFF/x-real-ip/socket), GET /api/roadmap (roadmap.ts: raw ROADMAP.md fetched from ROADMAP_URL — default the repo's main on GitHub — 60 s in-process cache, stale-on-error, 503 when cold+down, 404 when ROADMAP_URL=''), and three proxy mounts /api/{gateway,rag,orchestrator}/* (proxy.ts). proxy.ts strips the mount prefix, replaces any browser Authorization with `Bearer <platform key>`, propagates/creates x-request-id, strips hop-by-hop + content-encoding headers, binds the client abort signal to the upstream fetch, and streams bodies (SSE included) unbuffered. config.ts: env with localhost defaults, keys default 'demo-key'. server.ts is the only side-effectful entry (tsx runtime, no build step).
- `web/src/app` — FSD app layer: main.tsx (calls bootstrapTheme() before render), providers/AppProviders.tsx (React Query + i18next + Radix Tooltip.Provider), router/router.tsx (TanStack Router; '/' redirects to /models; routes /models /research /usage /knowledge /telemetry /mission), styles/tokens.css design tokens with light/dark via data-theme attribute.
- `web/src/pages` — One thin page per route (knowledge, mission-control, models, research, telemetry, usage) — each just composes its widget.
- `web/src/widgets` — Section UIs: research-console (TraceTimeline, AnswerView with clickable [n] citations, EvidenceList), model-catalog, usage-dashboard (Recharts), knowledge-manager (+StatsPanel), telemetry-explorer (model-runs + research-runs panels with cursor paging), mission-control (ServiceHealthTile polling + RoadmapPanel rendering docs/ROADMAP.md parsed at build time in model/roadmap.ts), app-nav.
- `web/src/features` — User actions: ask-research (useResearchStream owns the AbortController and folds SSE events into ResearchRunState), ping-model, select-model (zustand), switch-theme (zustand themeStore, persists console.theme, sets data-theme on <html>), switch-language, ingest-documents, search-knowledge.
- `web/src/entities` — Domain data: research (model/events.ts parses raw SSE frames to typed events; api/research.ts streamResearch + fetchResearchHistory), models, usage, model-run and research-run (normalize.ts projects snake_case wire rows to camelCase UI shapes), document (rag ingest/query/stats), service-health (probeService never throws — resolves to 'down'; useServiceHealth polls every 15s). Every fetch is Zod-parsed with the generated <op>200Schema.
- `web/src/shared` — shared/config/env.ts: API_BASE='/api', GATEWAY_BASE/ORCHESTRATOR_BASE/RAG_BASE all end in /v1, localStorage keys console.{theme,lang,token}. shared/api/client.ts (apiGet/apiPost, ApiError, optional Bearer from localStorage console.token), shared/api/sse.ts (streamSse over @microsoft/fetch-event-source: swallows heartbeats/empty frames, converts library auto-retry into a single fatal error, abort resolves quietly). shared/i18n (typed en/ru locales), shared/lib/markdown.ts (tiny parser with first-class [n] citation nodes), shared/ui (Radix-skinned Badge/Button/Card/Spinner/toggles, each with .module.css).
- `web/src/test + web/e2e` — Unit/component tests (Vitest jsdom, includes an axe a11y pass in a11y.test.tsx, render.tsx mirrors AppProviders) live under src/test; Playwright+axe live-browser smoke lives in e2e/ (separate runner, needs the compose stack).
- `infra` — bff.Dockerfile (node:26-slim, installs pnpm@11.15.1 explicitly since Node 26 has no corepack, sets verifyDepsBeforeRun=false, runs via tsx), web.Dockerfile (two-stage: pnpm build incl. `contracts generate` → static dist baked into caddy:2-alpine), Caddyfile (serves SPA with try_files fallback, reverse-proxies /api/* to bff:8787 with flush_interval -1 for SSE, strict CSP + security headers).
- `.github/workflows` — ci.yml: build job (contracts-in-sync git diff check, Biome, Steiger FSD, tsc -r, vitest -r, vite build) + e2e job (clones the 4 backend repos from github.com/INTERpol21 as siblings, docker compose up --wait, scripts/platform_smoke.py, then Playwright+axe). tag-release.yml: tags v<version> when package.json version changes on main, requires a matching '## [x.y.z]' CHANGELOG.md section, release notes are that section verbatim.
- `scripts/platform_smoke.py` — Cross-service smoke (httpx only): gateway completion+cache-hit, rag ingest→cited query, orchestrator /v1/research end-to-end (asserts plan|execute|reflect|synthesize in trace, rag in evidence sources, [1] in answer) and SSE stream. Two modes: umbrella (through Caddy+BFF at :8080/api) or --direct with SMOKE_*_PORT overrides.
- `docs/adr` — 11 platform-wide ADRs: polyrepo+contracts, single Postgres w/ schemas, LangGraph Postgres checkpointer, FSD, backend layered skeleton, Kubb codegen, unified /v1 prefix, BFF single origin, untrusted-context fencing, CSS Modules not Tailwind, no minimum release age.

## Invariants (breaking these breaks the platform)
- packages/contracts/src/{gateway,rag,orchestrator}/ is Kubb-generated but COMMITTED; CI runs `pnpm --filter @console/contracts generate` and fails on `git diff` — never hand-edit it, and never edit openapi/*.json without regenerating in the same commit. Kubb output.clean:true deletes the whole src/<service> dir on regen.
- All three backends serve their API under /v1; the SPA base URLs (GATEWAY_BASE/RAG_BASE/ORCHESTRATOR_BASE in web/src/shared/config/env.ts) all embed /v1, and the BFF strips ONLY the /api/<svc> mount prefix — /v1 travels upstream. Health probes are the deliberate exception: /healthz has no /v1.
- Single-origin, zero-CORS model: backends have no CORS and no browser-safe keys. Platform bearer keys exist only in BFF env (GATEWAY_KEY/RAG_KEY/ORCH_KEY, default 'demo-key' = the backends' offline default); proxy.ts drops any browser-supplied Authorization and injects them. Nothing in web/ may call a backend origin directly — everything goes through /api.
- SSE must never be buffered anywhere on the path: Caddyfile sets flush_interval -1, the BFF returns upstream.body unbuffered and sets x-accel-buffering:no, the Vite dev proxy sets x-accel-buffering:no. Adding any buffering middleware breaks the live trace.
- pnpm version is pinned in exactly one place — `packageManager` in root package.json (pnpm@11.15.1); CI's pnpm/action-setup has no version field and both Dockerfiles hard-code the same version (Node 26 ships without corepack). Change it in all three or builds diverge.
- pnpm workspace policy lives in pnpm-workspace.yaml, not .npmrc/package.json: onlyBuiltDependencies (biome, esbuild need postinstall binaries), allowBuilds must answer yes/no for every package pnpm asks about (otherwise `pnpm install` appends an allowBuilds: stub that breaks installs if committed), strictDepBuilds:false, and minimumReleaseAge:0 (ADR-0011 — deliberately disabled 24h quarantine because same-day releases blocked builds).
- TypeScript 7 (native tsgo): the JS compiler API is gone, so any tool loading a .ts config via cosmiconfig fails — that is why steiger.config is .js. Also baseUrl was removed from the language; web/tsconfig.json uses `paths` relative to the file instead.
- FSD import direction (app → pages → widgets → features → entities → shared) is CI-enforced by Steiger; slices are consumed only via their index.ts public API (shared is segment-only, fsd/public-api off there). Cross-layer or deep imports fail `pnpm --filter @console/web lint:fsd`.
- Version bump on main IS the release: tag-release.yml tags v<version> from package.json but hard-fails unless CHANGELOG.md contains a matching '## [x.y.z]' section; release notes are extracted verbatim from that section.
- i18n locales are typed TS modules (en.ts/ru.ts); a missing key is a typecheck failure, not a runtime fallback. Both languages must be updated together.
- The e2e/compose world assumes the four backend repos are checked out as siblings (../llm-gateway, ../rag-pgvector, ../mcp-tools-server, ../agent-orchestrator); compose builds them from source. CI clones them from github.com/INTERpol21.
- BFF has no build/bundle step: `start` is `node --import tsx src/server.ts`, imports use explicit .ts extensions (allowImportingTsExtensions), and the Docker image ships source + node_modules. tsc is typecheck-only (`--noEmit`) across the whole monorepo.

## Gotchas
- Backends are NOT published on the host in the umbrella stack — only Caddy :8080 is. `curl localhost:8081` hits nothing (or, worse, some other local process). Use /api/<svc>/... through :8080, or run backends by hand and use platform_smoke.py --direct. Known local quirk: if another dev server owns 8080-8083, compose may publish on IPv6 only and 127.0.0.1 silently reaches the other process (see the SMOKE_*_PORT overrides).
- Every backend's own unit suite mocks its neighbours, so cross-service breakage ships green: a missing /v1 prefix in the orchestrator's RAG client once passed ruff/mypy/pytest/CodeQL while every retrieval 404'd and the agent answered with no evidence. scripts/platform_smoke.py is the only gate that exercises real service-to-service links — run it after touching any client base URL.
- parseResearchEvent: the orchestrator's real wire format is unnamed 'message' events with {type: ...} inside the JSON — the named-event branch (trace/evidence/token/...) is a fallback for other producers. If you only test named events, the real stream will parse to null. Unknown/'custom' frames and heartbeats are silently dropped by design.
- Kubb regen is destructive (output clean:true): anything hand-placed under packages/contracts/src/<service>/ is deleted by `pnpm contracts`. Also web.Dockerfile reruns `contracts generate` inside the image build, so a stale openapi/*.json silently ships stale types even if you edited generated files locally.
- pnpm install can mutate pnpm-workspace.yaml: a new transitive package with a build script makes pnpm append an `allowBuilds:` stub for a human to fill in — committing that stub as-is breaks installs. Answer yes/no explicitly in the allowBuilds map instead.
- TS7/tsgo traps: renaming steiger.config.js back to .ts breaks Steiger (cosmiconfig needs the removed JS compiler API); adding baseUrl to a tsconfig is a hard error. Expect the same failure mode in any tool that programmatically reads tsconfig.
- The BFF rate limiter is per-process and trusts x-forwarded-for (first hop) for bucketing — fine behind Caddy, but a client varying XFF gets fresh buckets (idle sweep bounds the map). Tests pin 429 + Retry-After behaviour; /api/health is registered BEFORE the limiter and must stay free.
- probeService never throws — health failures become {status:'down'}. Don't 'fix' it to throw; the mission-control board renders the failure. Conversely apiGet DOES throw ApiError on non-2xx.
- shared/api client and sse read an optional bearer from localStorage 'console.token' and send it to the BFF — but the BFF strips and replaces browser Authorization. That token is not what authenticates against backends; don't build auth assumptions on it.
- Vitest only picks up src/**/*.{test,spec}.*; a Playwright spec written under src/ (or a Vitest test under e2e/) will run under the wrong runner. e2e also assumes Chromium is pre-provisioned (PLAYWRIGHT_BROWSERS_PATH) — CI installs it explicitly.
- .gitignore ignores web/src/shared/api/generated/ (a legacy location) — but contract codegen actually lives in packages/contracts/src and IS committed. Don't conclude generated code is untracked.
- RoadmapPanel renders docs/ROADMAP.md LIVE from the main branch (useLiveRoadmap: /api/roadmap through the BFF, refetch every 60 s) with the build-time `?raw` copy as fallback whenever the live fetch fails or parses empty (data-live attr says which source won). Checkbox `- [~]` = in-progress (accent badge). The parser relies on the roadmap's conventions (## sections, `- [x]`/`- [ ]` items, `**bold title.**`, `~~strikethrough~~`); web/src/test/roadmap-parse.test.ts pins them, so format changes to ROADMAP.md can fail web tests. Item text is deliberately not translated — it renders the repo document.
- jsdom lacks scrollIntoView; citation code calls it, and web/src/test/setup.ts stubs it. New DOM APIs used in components may need the same treatment or component tests crash.
- bff.Dockerfile sets verifyDepsBeforeRun=false because pnpm's runtime lockfile re-check can crash-loop the container; removing that line reintroduces the crash-loop risk on policy mismatch.
- The e2e CI job needs Docker Hub reachable and the four sibling repos clonable; the unit `build` job needs neither. Per README, running the e2e job is the open M5 item — do not assume it has been passing.

## Key flows
- Prod request path: browser → Caddy :8080 (single origin; serves SPA, /api/* → bff:8787 with flush_interval -1) → BFF (rate limit, strip /api/<svc>, inject Bearer key, add x-request-id) → gateway:8080 | rag:8081 | orchestrator:8083, all under /v1 upstream. Dev path: Vite :5173 proxies /api → BFF :8787 (pnpm dev:bff), same from there.
- Research SSE: QuestionForm → useResearchStream (AbortController) → streamResearch → streamSse POST /api/orchestrator/v1/research/stream → parseResearchEvent maps frames → applyEvent folds into ResearchRunState → TraceTimeline/AnswerView. The orchestrator sends UNNAMED SSE frames with the kind inside the JSON payload ({type:'trace'|'result'|'error'}); named-event frames (trace/evidence/token/answer/done) are a tolerated fallback; 'custom' frames are dropped.
- Typed REST: entity api/*.ts builds URL from {GATEWAY,RAG,ORCHESTRATOR}_BASE (all /api/<svc>/v1) → apiGet/apiPost → Zod .parse with the Kubb-generated response schema → optional normalize.ts (snake_case wire → camelCase UI) → React Query hook → widget.
- Contract regen: backend repo exports create_app().openapi() to JSON → copy into packages/contracts/openapi/<service>.json → `pnpm contracts` (Kubb, clean:true wipes src/<service>) → commit both; CI diffs regenerated output against the committed tree.
- Health board: probeService fetches /api/{gateway,rag,orchestrator}/healthz and /api/health (BFF) — unversioned, no /v1 — every 15s via React Query; failures resolve to status 'down', never throw.
- Umbrella compose: gateway needs redis; rag needs postgres+gateway; orchestrator needs gateway+rag+mcp+postgres (LangGraph checkpoints in schema 'orchestrator', ORCH_CHECKPOINT_SCHEMA); bff waits for all three HTTP backends healthy; caddy waits for bff. Ollama only under `--profile full`.

## Conventions
- Strict FSD layout in web/src: <layer>/<slice>/{api,model,ui}/ with an index.ts public API per slice; pages are thin wrappers around widgets; state via zustand (theme, model selection) or React Query (server data), never both for the same data.
- CSS Modules + design tokens only (web/src/app/styles/tokens.css); no Tailwind, no component kits — Radix primitives skinned with co-located *.module.css (ADR-0010). Light/dark via data-theme on <html> plus prefers-color-scheme.
- Biome 2 is the single linter/formatter (single quotes, trailing commas, 100-col, 2-space); generated contract dirs and openapi/ are excluded in biome.json. Pre-commit hooks run biome --write, steiger, and pnpm -r typecheck.
- Every API response crossing the wire is runtime-validated with the generated Zod schema (<operation>200Schema.parse) before use; snake_case wire fields are projected to camelCase in entities/*/model/normalize.ts, never used raw in UI.
- Prefer subpath contract imports (@console/contracts/gateway) over the namespaced root barrel (root barrel exists for tooling only).
- Unit/component tests live in web/src/test/*.test.{ts,tsx} and run under Vitest jsdom (vitest include is src/** only); Playwright specs live in web/e2e/ and are driven by E2E_BASE_URL (default http://localhost:8080). BFF tests drive the Hono app via app.request() with a mocked global fetch — no sockets.
- React Query keys are structured factories per entity (modelKeys.catalog(), healthKeys.service(id)).
- Comments in configs/Dockerfiles explain WHY a setting exists (this repo leans on that heavily — preserve the style when editing).

## Test strategy
Four layers. (1) Unit/component: `pnpm -r test` — BFF Vitest suite (bff/src/bff.test.ts, rate-limit.test.ts) drives the Hono app via app.request() with mocked fetch, pinning prefix-stripping, Authorization replacement, SSE pass-through headers, x-request-id echo, abort propagation, 429s; web Vitest (jsdom, globals, setup in web/src/test/setup.ts, provider wrapper in render.tsx) covers widgets, SSE/event parsing, format/i18n, plus a fast axe a11y pass — no services needed. (2) Static gates: Biome, Steiger FSD boundaries, `pnpm -r typecheck` (tsgo), and the contracts-in-sync git-diff check, all in the CI `build` job and mirrored by pre-commit hooks. (3) Cross-service: scripts/platform_smoke.py against a running stack — the only thing testing real inter-service wiring. (4) Live-browser e2e: Playwright + @axe-core/playwright in web/e2e/smoke.spec.ts against the compose stack via E2E_BASE_URL (boot → route → streamed research answer → serious/critical axe violations = []), run in the separate CI `e2e` job which brings the stack up with `docker compose up -d --build --wait`. Local totals ~31s for the console gates (docs/BUILD_TIMES.md).
