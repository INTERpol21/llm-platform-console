# Карта проекта — история, решения, уроки

Живой документ-карта AI-платформы: что построено, когда, зачем и какой ценой.
Читается сверху вниз как история; каждый закрытый пункт несёт **дату** и
**версию релиза**, чтобы можно было вспомнить и сверить с тегами GitHub.
Правило ведения: закрыл пункт — допиши сюда дату, версию, «что/сколько/зачем»
и урок, если он был. Актуальный план — в [ROADMAP.md](ROADMAP.md) (англ., его
рендерит Mission control); этот файл — память проекта по-русски.

## Что это за платформа

Портфолио-платформа из 5 репозиториев под `github.com/INTERpol21`. Связанная
мини-платформа для работы с LLM, целиком работающая **офлайн** (mock-провайдер,
hash-эмбеддинги, стаб-поиск) — реальные провайдеры подключаются конфигом.

| Репозиторий | Роль | Порт | Версия |
|---|---|---|---|
| llm-gateway | OpenAI-совместимый шлюз: роутинг, фолбэки, кэш, лимиты, стоимость | 8080 | 1.1.0 |
| rag-pgvector | RAG на pgvector: гибридный поиск (вектор+BM25/FTS через RRF), реранкер, цитаты | 8081 | 1.3.0 |
| mcp-tools-server | MCP-инструменты: офлайн-поиск, read-only SQL, файловый sandbox | 8082 | 1.1.0 |
| agent-orchestrator | LangGraph-агент: план → исполнение → рефлексия → синтез | 8083 | 1.2.1 |
| llm-platform-console | Консоль: React 19 + Hono BFF + Caddy, зонтичный compose всей платформы | 8080 (Caddy) | 1.1.0 |

Топология: браузер → Caddy (один origin) → BFF `/api/*` → {gateway, rag,
orchestrator}; orchestrator → {gateway, rag, mcp}; rag → gateway (синтез и
эмбеддинги). Все API под `/v1`; ключи живут только в BFF.

## Эпоха M1 — Объяснитель ценности *(облачная работа, до 2026-07-20)*

**Зачем:** прежде чем писать код — зафиксировать, что платформа делает и чем
отличается; артефакт, который можно показать.

**Что сделано:** самодостаточный explainer платформы (что/зачем/как связано).

## Эпоха M2 — DX-базлайн и рефактор скелета *(облачная работа, до 2026-07-20)*

**Зачем:** четыре бэкенда должны выглядеть и собираться одинаково, чтобы
дальше каждый новый инструмент подключался один раз и сразу везде.

**Что сделано (во всех 4 Python-репо):**
- uv + uv.lock как источник правды; `requirements.txt` только экспортируется
  (`make lock`), Docker ставит из него.
- mypy strict, ruff (E/F/I/W/B/UP), pre-commit.
- Слоистый скелет: routes тонкие, логика в services, фреймворк-независимая.
- Non-root Docker + HEALTHCHECK; JSON-логи с `X-Request-ID`.
- Security-CI: pip-audit, bandit, CodeQL, Dependabot.

**Решение:** polyrepo + пакет контрактов, а не монорепо (ADR-0001).

## Эпоха M3 — Фичи бэкендов *(облачная работа, до 2026-07-20)*

**Зачем:** превратить скелеты в работающие сервисы с реальными механиками.

**Что сделано:**
- gateway: каталог моделей + ping без токенов, китайские провайдеры
  (DeepSeek/Moonshot/Qwen) + Ollama, circuit breaker, телеметрия `model_runs`.
- orchestrator: SSE `/v1/research/stream`, Postgres/Memory checkpointer,
  `/history`, идемпотентность.
- rag: HNSW-тюнинг, dim-guard, local-first приоритеты источников, загрузка
  файлов md/txt/pdf/docx.
- Экранирование prompt-injection во всех сервисах (OWASP LLM01/08) + promptfoo
  red-team гейт на границе синтеза rag.

**Решения:** один Postgres со схемами на сервис (ADR-0002), LangGraph
Postgres-checkpointer (ADR-0003), фенсинг недоверенного контекста (ADR-0009).

## Эпоха M4 — Консоль и зонтичный compose *(облачная работа, до 2026-07-20)*

**Зачем:** платформе нужно лицо и один вход; браузер не должен знать ключей.

**Что сделано:** консоль на 6 разделов, строгий FSD (Steiger), CSS Modules,
i18n RU/EN, Kubb-контракты из OpenAPI-снапшотов, Hono BFF (подмена ключей,
rate-limit, SSE без буферизации), umbrella docker-compose на 8 контейнеров за
одним origin.

**Решения:** FSD (ADR-0004), Kubb-кодоген (ADR-0006), единый `/v1` (ADR-0007),
BFF как единственный origin (ADR-0008), CSS Modules вместо Tailwind (ADR-0010).

## Эпоха M5 — Полный CI и ADR *(2026-07-20 … 2026-07-21)*

**Зачем:** зелёный main как норма; решения — записаны, а не в голове.

**Что сделано:** jsdom+axe a11y-гейт, Playwright e2e-каркас, все `ci.yml`
зелёные; 11 ADR. **2026-07-20/21 — v1.0.0 во всех пяти репо.**

Тогда же (2026-07-20): gateway `POST /v1/embeddings` (общая цепочка ретраев и
фолбэков с completions), CHANGELOG-и везде, кросс-сервисный smoke в CI консоли,
гейт дрейфа `requirements.txt` ↔ `uv.lock`.

**Решение:** снята 24-часовая карантин-задержка pnpm (ADR-0011) — осознанный
размен: свежие релизы в день выхода против supply-chain паранойи; компенсация
позже пришла в виде Trivy-гейта.

## Принятые решения (ADR, кратко по-русски)

| ADR | Решение | Почему |
|---|---|---|
| 0001 | Polyrepo + пакет контрактов | Каждый сервис — самостоятельный артефакт портфолио |
| 0002 | Один Postgres, схемы на сервис | Демо-масштаб: изоляция без зоопарка инстансов |
| 0003 | LangGraph Postgres-checkpointer | Возобновляемые сессии и аудит из коробки |
| 0004 | FSD для фронта | Принудительная направленность импортов, CI-гейт Steiger |
| 0005 | Слоистый скелет бэкендов | routes тонкие / services чистые / клиенты типизированные |
| 0006 | Kubb-кодоген из OpenAPI | Типы и Zod из реальных снапшотов; дрейф ловит CI |
| 0007 | Единый префикс `/v1` | Один контрактный пакет, одна форма путей |
| 0008 | BFF — единственный origin | Ключи не попадают в браузер; ноль CORS |
| 0009 | Фенсинг недоверенного контекста | OWASP LLM01/08; promptfoo-гейт |
| 0010 | CSS Modules, не Tailwind | Токены + co-located стили, без утилитарного шума |
| 0011 | Отключён карантин релизов pnpm | Свежие пакеты в день выхода; компенсация — Trivy |

## Ошибки и уроки (чтобы не наступать повторно)

1. **zsh и `$img:latest`** — модификатор `:l` молча съедает двоеточие; всегда
   `${img}:latest` в скриптах.
2. **`wait <pid>` между шагами GitHub Actions не работает** — шаги в разных
   шеллах; фоновая сборка должна писать exit-код в файл.
3. **COMPOSE_BAKE может молча не делегировать** — явный `docker buildx bake`
   c `--allow fs.read=..` и явными `image:` в overlay.
4. **pnpm сам дописывает `allowBuilds`-заглушку** в pnpm-workspace.yaml — если
   закоммитить её как есть, установка ломается; отвечать yes/no явно.
5. **npm внутри node-образа — источник CVE** (vendored node_modules): если
   runtime зовёт только pnpm, npm из образа удалить.
6. **Eval-гейт не заметил отсутствие стемминга** — одинокий русский документ
   выигрывал на любом кириллическом пересечении; настоящие гарды — прямые
   тесты (textnorm, pgvector-дистрактор). Метрика ≠ регрессионный тест.
7. **anyio заворачивает ошибки в ExceptionGroup** — «unhandled errors in a
   TaskGroup» в трейсе бесполезен; разворачивать до листовых причин.
8. **`docker compose up --build` не всегда пересоздаёт контейнер** — после
   пересборки проверять, что контейнер реально отдаёт новый бандл
   (`grep` по `/srv/assets`), при сомнении `--force-recreate`.
9. **Порядок мержа связки клиент/сервер** — сначала сторона, которую старый
   визави игнорирует (клиент с лишним заголовком), потом требующая сторона.
10. **Список «сделано» без даты и версии забывается за неделю** — потому
    каждый закрытый пункт в этом файле и в ROADMAP несёт дату и релиз.
11. **Кросс-сервисные поломки зелёные в юнитах** — каждый сервис мокает
    соседей; единственный честный гейт — `platform_smoke.py` на живом стеке.
12. **Незакоммиченная сгенерённая правка тихо уезжает** — генерённые контракты
    коммитятся, CI перегенерирует и падает на diff; это спасало не раз.

## Эпохи впереди (заведены 2026-07-24, план — в ROADMAP)

- **M7 — Design (отдельный трек):** редизайн консоли в «спокойный операторский
  минимализм» + функциональные дыры Knowledge UI. Ведётся отдельно от
  бэкенд-работы. Первый закрытый пункт трека — **живой Delivery
  roadmap** (консоль 1.2.0, 2026-07-24): панель читает ROADMAP.md прямо из
  main через BFF (обновление раз в минуту), маркер `- [~]` показывает «в
  работе прямо сейчас» — состояние плана видно онлайн, без пересборки.
- Первое движение по M8 (2026-07-24): пункт (a) «durable orchestrator в
  umbrella» закрыт как **уже верный** — аудит-агент ошибся, `ORCH_DATABASE_URL`
  в compose задан давно; проверено рестартом контейнера (история треда
  выжила). Урок: выводы агентов сверять с живой системой перед планированием.
- **M8 — Scale-out (крупный бэкенд-трек):** вынести пер-процессное состояние
  (лимитер/брейкер/кэш шлюза, лимитер BFF) в Redis, реплики за балансировщиком,
  вертикальные ручки (workers, пулы), нагрузочные тесты, ускорение CI и
  Docker-сборок. Детали и чекбоксы — в ROADMAP.
