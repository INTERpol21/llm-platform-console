# Гибридный поиск: вектор + BM25 через RRF

Платформа ищет двумя способами одновременно. Векторная ветка сравнивает
эмбеддинги по косинусной близости — она находит смысл, даже когда слова не
совпадают. Ключевая ветка (Okapi BM25 в памяти, Postgres FTS в pgvector)
ловит точные термины и редкие идентификаторы.

Результаты сливает Reciprocal Rank Fusion: каждый документ получает
1/(k + ранг) из каждой ветки, суммы сортируются. RRF не требует калибровки
скорингов между ветками — только ранги, поэтому он устойчив к разной
природе оценок.

Русские и английские запросы стеммируются Snowball-стеммером в обеих ветках:
запрос «о поиске» находит документ со словом «поиск».
