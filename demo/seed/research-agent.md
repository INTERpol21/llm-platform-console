# Research-агент: план → исполнение → рефлексия → синтез

Оркестратор строит план из 1–3 шагов (поиск в базе знаний, веб-поиск через
MCP), исполняет независимые шаги параллельно через asyncio.gather, затем
рефлексия решает, хватает ли доказательств, и при нужде добавляет один
дополнительный шаг. Синтез пишет ответ с точечными цитатами [n].

Каждый шаг деградирует отдельно: упавший клиент помечает свой шаг как FAILED,
остальные доносят доказательства, и ран всегда завершается ответом, а не
ошибкой 500. История тредов хранится в Postgres-чекпоинтере и переживает
рестарты.
