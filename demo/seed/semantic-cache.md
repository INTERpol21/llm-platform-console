# Семантический кэш ответов

Точный кэш помогает только при побайтовом совпадении запроса. Семантический
слой эмбеддит промпт и отдаёт сохранённый ответ, когда новый вопрос близок по
смыслу: «как работает RRF» переиспользует ответ на «объясни reciprocal rank
fusion».

Гардрейлы против почти-подходящего ответа: только temperature=0, партиции по
модели и области кэша, порог косинусной близости, ограниченный FIFO. Сбой
эмбеддинга просто пропускает кэш — completion никогда не падает из-за него.
